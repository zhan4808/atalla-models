## 1. Architecture Overview

```
PyTorch nn.Module
    │
    ├─ atalla-graph/        FX graph capture, op normalization, tiling, code generation
    ├─ aihw-ppci-compiler/  AtallaC → assembly compiler
    └─ functional_sim/      Assembler, .in builder, emulator
```

**Two execution paths:**

```
PATH 1: HAND-WRITTEN ASSEMBLY (ground truth, standalone validation)
  build_relu.py ──► encode + DRAMWriter ──► complete .in ──► functional_sim

PATH 2: C COMPILER (primary graph pipeline)
  c_emitter.py ──► .c kernel ──► ppci ──► .s ──► build_compiler ──► instruction section
  DRAMWriter ──────────────────────────────────────────────────────► data section
                                                                         │
                                                            render_testfile() → .in → functional_sim
```

`build_compiler.py` produces the instruction section. The data section (config at `0x3C` + tensors) is generated by `c_emitter.py`'s `DRAMWriter`. `render_testfile()` combines both into a `.in` file.

**Passthrough ops** (`flatten`, `transpose`, `dropout`) need no kernel — they're pure tensor reshapes with no computation. The Python orchestrator just reshapes the numpy array in `activation_cache` and passes it to the next layer. No `.in` file is generated.

**Current status:** Only `mul` (BasicModule residual) still falls back to NumPy. **AlexNetSmall** at default `scale` uses **zero** NumPy compute fallbacks (passthrough reshapes only).

## 2. atalla-graph

### 2.1 Graph Capture (`graph/fx_capture.py`)

1. Converts weights to `bfloat16`
2. `torch.fx.symbolic_trace(model)` → flat FX graph
3. `ShapeProp` propagates shapes
4. Tags each node with `node.meta["atalla_op"]` via lookup tables:

| Source | Example | Maps to |
|--------|---------|---------|
| `_OP_MAP` (call_function) | `torch.matmul`, `F.relu`, `F.conv2d` | `"matmul"`, `"relu"`, `"conv"` |
| `_METHOD_MAP` (call_method) | `.flatten()`, `.add()`, `.transpose()` | `"flatten"`, `"add"`, `"flatten"` |
| `_MODULE_MAP` (call_module) | `nn.Conv2d`, `nn.MaxPool2d` | `"conv"`, `"maxpool"` |

`torch.matmul` and `torch.ops.aten.matmul.default` both map to `"matmul"` — the aten variant appears when `lower_linear_modules` decomposes `nn.Linear` into `transpose + matmul + add`.

### 2.2 Tile Planning (`graph/tile_planner.py`)

Hardware: VL=32, systolic tile=32×32, scratchpad=32 slots/bank, bf16.

| Op | Planning |
|----|----------|
| **Conv** | im2col: M=Ho×Wo, K=R×S×C_in, N=C_out, tiles=ceil(dim/32) |
| **Linear/Matmul** | M, K, N from shapes, tiles=ceil(dim/32) |
| **ReLU/Add/Mul** | total_elements, width=min(total, 32) |
| **Softmax** | length from last dimension |
| **MaxPool** | H, W, C, pool_size, stride → H_out, W_out |
| **Flatten/Transpose** | Passthrough, no tiling |

DRAM addresses start at `0x1000`, 2KB-aligned. `0x00`–`0x3C` reserved for ADDR_TABLE.

### 2.3 Code Generation (`codegen/c_emitter.py`)

`emit_node()` dispatch:

| Op | Handler | Backend |
|----|---------|---------|
| conv, linear, matmul | `emit_conv/linear/matmul` | On-chip (GEMM kernel) |
| relu | `emit_relu` | On-chip |
| softmax | `emit_softmax` | On-chip |
| maxpool | `emit_maxpool` | On-chip (vertical max) |
| add | `emit_add` | On-chip (dual scratchpad) |
| mul | `emit_mul` | NumPy fallback |
| adaptive_avg_pool | `emit_adaptive_avg_pool` | NumPy fallback |

Each `emit_*` returns a `LayerEmission` with: `c_source` (AtallaC), `dram` (DRAMWriter with config + tensors), `output_addr`, `output_elements`.

**Kernel generators** in `atalla-graph/kernels/`:

| Generator | Technique |
|-----------|-----------|
| `gemm.py` | Tiled GEMM: `scpad_load/store`, `vector_load/store`, `load_weights`, `gemm()` |
| `relu.py` | `make_mask("<", v, zero)` → `vec_op_masked("*", v, 0.0, mask)` |
| `softmax.py` | `RMAX` → shift → `EXP` → `RSUM` → `1.0 / sum` |
| `maxpool.py` | `make_mask(">")` → blend (vertical max) |
| `add.py` | Dual scratchpad: A→SP0, B→SP1, `vec_op_masked("+")` |

All use first-class intrinsics. Only `lw_s`, `li_s`, `halt` remain as inline asm.

### 2.4 `run_graph.py`

CLI: `--model {basic,alexnet_small}`, `--mode {validate,schedule,both}` (default `both`), `--out-dir` (default `out/graph`, **gitignored**), `--scale`, `--export-kernel-bundle`, `--quiet`.

- **`validate`:** compile → `build_compiler` → `.in` → emulator vs FX reference (cosine / max diff per node).
- **`schedule`:** Vihaan's `generate_schedule` path (C-focused export).
- **`both`:** runs validate then schedule.

## 3. aihw-ppci-compiler

### 3.1 AtallaC Extensions

| Feature | Syntax |
|---------|--------|
| Vector type | `vec v;` (32-element register) |
| GEMM | `gemm(a_row, c_row, mask)` |
| Masked ops | `vec_op_masked("+", a, b, mask)` |
| Mask creation | `make_mask(">", a, b, mask)` |
| Intrinsics | `scpad_load/store`, `vector_load/store`, `load_weights`, `sqrt` |
| Implicit division | `1.0 / x` → `rcp_bf + mul_bf` |
| Type casts | `float(x)` → `stbf_s`, `(int)y` → `bfts_s` |

### 3.2 Compilation Flow

```
.c → ppci frontend → IR → instruction selection → register allocation → .s
.s → build_compiler: parse → bootstrap → expand li_s → VLIW schedule → encode → .in
```

### 3.3 Limitations

- **Stack frame overflow**: Many live vectors exceed 12-bit offset range. Workaround: reduce live vector count.
- **No vector SQRT / shift.vi**: Only scalar `sqrt()` works. Blocks vector-width ops.
- **Config loads require inline asm**: No intrinsic for `lw_s`, `li_s`, `halt`.

## 4. functional_sim

### 4.1 Architecture

| Component | Description |
|-----------|-------------|
| Scalar regs | 34 regs; x0=zero, x2=stack ptr, x33=vector spill / GMEM boundary helper |
| Mask regs | 16 predicate regs (**m0 is writable**, not hardwired); used with masked vector ops |
| Vector regs | 64 regs × 32 BF16 lanes (compiler/emulator convention: BF16 in lower half of float word) |
| Scratchpads | SP0, SP1: 32 rows × 32 BF16 elements (tile semantics per ISA) |
| Memory | GMEM + scratchpad models; linear BF16 tensor regions vs compiler spill slots (see `scpad_ls.py`) |

### 4.2 Encoding Toolchain

**`build.py`** (Sahil's): `encode_instruction` → `assemble_file` → `emit_test_format` (1 instr/packet, not dependency-aware) + `DRAMWriter` + `render_testfile`. Used by `build_*.py` scripts.

**`build_compiler.py`**: `compile_asm()` — parse → bootstrap → li_s expansion → **dependency-aware VLIW scheduling** → encode. Packs multiple instructions per packet. Used by the graph pipeline.

Both produce the same `.in` format. `build_compiler` achieves better slot utilization.

### 4.3 ADDR_TABLE Convention

Kernels read parameters from address **0x3C**:

| Layout | Fields |
|--------|--------|
| GEMM/Conv/Linear | A_GMEM, W_GMEM, C_GMEM, M, N, K, tile counts |
| ReLU/Softmax | IN_GMEM, OUT_GMEM |
| Add | A_GMEM, B_GMEM, C_GMEM |

### 4.4 Emulator / toolchain alignment (high level)

| Area | Notes |
|------|--------|
| Mask RF | m0 and siblings modeled as real predicate registers (not stuck at all-ones). |
| BF16 in vector regs | Load/store paths match compiler packing (lower 16 bits of float word). |
| GMEM vector loads (`vreg.ld` / spills) | Spill window vs BF16 tensor linear access (x33 + address rules). |
| `gemm.vv` / `lw.vi` | Systolic weight buffer reset between tiles; ordering matches tiled GEMM. |
| Stack vs tensors | Stack pointer placed above DRAM image so stack growth does not clobber weights/activations. |
| SDMA | Three scalar operands: base addresses + packed metadata word (`emit_sdma_metadata_asm` in `build.py`). |
| `build_compiler.py` | VLIW schedule + encode for compiler-emitted `.s`; paired with `instruction_latency.py`. |

## 5. Kernel Library

Three levels, validated bottom-up:

| Level | Location | Purpose |
|-------|----------|---------|
| Ground truth | `functional_sim/build_*.py` | Hand-written asm + DRAMWriter data, run on emulator |
| Reference C | `aihw-ppci-compiler/atalla_tests/kernels/*.c` | AtallaC with intrinsics, compiled via ppci |
| Generators | `atalla-graph/kernels/*.py` | Parameterized AtallaC for arbitrary tensor shapes |

### GEMM Algorithm (used by conv, linear, matmul)

```
for each tile triple (mi, ni, ki):
  scpad.ld A_tile, W_tile, C_tile
  for w in 0..TILE: lw.vi w          # preload weights into systolic array
  for r in 0..TILE:
    vreg.ld a_row, c_row
    gemm.vv c_row, a_row, mask       # c += a * W
    vreg.st c_row
  scpad.st C_tile
```

## 6. Full Pipeline Integration

### 6.1 Execution Flow

```python
gm = build_graph(model, input)  # FX trace → lower_linear_modules → ShapeProp → normalize → tile plan

for node in gm.graph.nodes:
    if passthrough(node):                    # flatten, transpose, dropout
        activation_cache[node.name] = reshape(prev)
        continue

    emission = emit_node(node, gm, activation_cache)

    if emission.skip_emulator:               # NumPy fallback
        activation_cache[node.name] = emission.numpy_result
    else:                                    # on-chip
        compile_and_assemble(emission)       # .c → ppci → .s → build_compiler → instr section
        write(.in = instr_section + DRAMWriter_data)
        mem = run_emulator(.in)
        activation_cache[node.name] = read_bf16(mem, output_addr)

compare(activation_cache[final], pytorch_reference)
```

Each layer is a standalone emulator invocation with fresh state. Stack pointer (`x2`) placed dynamically above all DRAM data.

### 6.2 How `.data` Propagates

1. `emit_<op>()` creates a `DRAMWriter` → writes config at 0x3C + input/weight tensors as BF16
2. `compile_and_assemble()` produces the instruction section only (strips `.data` from `build_compiler` output)
3. `render_in_file()` in `c_emitter.py` wraps `functional_sim.build.render_testfile(instr, dram.render_data_mem())` → complete `.in`
4. Emulator loads `.in`, kernel reads config via `lw_s`, uses SDMA to move tiles between DRAM and scratchpad
5. `_read_bf16(mem, output_addr)` extracts output from emulator memory
6. Output goes into `activation_cache` → next layer's `emit_<op>()` serializes it as input into a fresh `DRAMWriter`

Validated and working for all kernels and full AlexNet (21 emulated nodes, 0 NumPy).

## 7. AlexNet Walkthrough

### AlexNetSmall (`model/alexnet_small.py`)

`scale` parameter controls channel widths: `sc(c) = max(1, int(c * scale))`. At `scale=0.01`: channels 64→1, 192→1, 384→3, 256→2. Input 32×32. No AdaptiveAvgPool or Dropout.

Vihaan's full AlexNet (`model/alexnet.py`) uses `torchvision.models.alexnet()`: 224×224 input, full channel counts, includes AdaptiveAvgPool2d and Dropout. All ops we support except AdaptiveAvgPool2d (which is identity for AlexNet's spatial dims) and Dropout (removed at inference).

### FX Graph (AlexNetSmall, scale=0.01)

After `lower_linear_modules`: **42 nodes** — 21 emulated (5 conv, 7 relu, 3 matmul, 3 maxpool, 3 add), 0 NumPy, 4 passthrough (flatten + transpose).

### Conv1 Example

`Conv2d(3, 1, 3×3, stride=1, pad=1)` on `(1, 3, 32, 32)`:
- im2col: M=1024, K=27, N=1 → `_gemm_c(1024, 1, 27)` → 32 tiles

## 8. Validation Results

The functional simulator is not cycle-accurate. For **poster / roofline metrics** (static vs dynamic VLIW slot efficiency, SDMA bytes moved, FLOP counters, arithmetic intensity, per-layer histograms), run `run_graph.py --mode validate --metrics-json …` and see **`README.md` § Performance metrics (poster / roofline)**.

### BasicModule (dim=32, depth=2)

Typical run: **9** emulated layers, **1** NumPy (`mul`), **1** passthrough; end-to-end cosine **≈0.99999** (BF16 + depth).

### AlexNetSmall (scale=0.01)

**21** emulated, **0** NumPy compute ops, **4** passthrough reshapes; end-to-end cosine **≈0.996** with accumulated BF16 error across conv + FC stack. Run `run_graph.py --mode validate` for exact metrics (`max_diff`, per-node cos).

- additive: `atalla_tests/kernels/` reference C kernels + validation scripts

### Emulator (`functional_sim`) — [atalla-functional-sim](https://github.com/Purdue-SoCET/atalla-functional-sim)

Developed in lockstep with this repo; consumed as a **submodule** at `functional_sim/` (no longer copied from the full `atalla` monorepo in-tree).

- **Memory / ISA fidelity:** BF16 GMEM linear access vs spill slots, mask register file, GEMM weight lifecycle, SDMA metadata, stack placement.
- **Tooling:** `build.py` (assembler + `DRAMWriter` + `emit_sdma_metadata_asm`), `build_compiler.py` (scheduler/encoder), `instruction_latency.py`, assorted `build_*.py` references, `validate_and_benchmark.py`, `validate_build_generators.py`.
- **Docs:** `ASSEMBLY_SYNTAX.md` (VM `vreg.ld`/`st` 5-operand form, 3-operand SDMA, etc.).

### Graph (`atalla-graph`)

- **Upstream spine** (Vihaan): `graph/export_fx.py`, `graph/lower_modules.py`, `graph/memoryallocator.py`, `scripts/generate_schedule.py`, parts of `model/`, `main.py`, legacy `kernels/kernels.c/h` (as present).

- **SoCET / models layer:** `graph/fx_capture.py`, `graph/tile_planner.py`, `codegen/c_emitter.py`, `codegen/dram_builder.py`, `kernels/*.py` (AtallaC generators), `run_graph.py`, `model/alexnet_small.py`, export scripts under `scripts/`.

- **Notable codegen behavior:** conv weight layout matches `im2col` patch order; optional conv bias broadcast into `C_GMEM`; conv output reshaped to NCHW for downstream nodes; BF16-aligned `activation_cache` for placeholders/weights; richer per-node validate stats in `run_graph.py`.

### Known Limitations

- compiler stack frame overflow (reduce live vector count)
- due to no vector sqrt / shift.vi -> use decompose + scalar sqrt , vertical-only maxpool
